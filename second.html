硬件技术栈

  • 微控制器 ESP32-WROOM-32
  • 温度传感器 DS18B20 (1-Wire)
  • 光照传感器 光敏电阻 + ADC
  • 显示屏 0.96" OLED I2C
  • 继电器 5V 10A/250VAC

软件技术栈

  • 开发框架 Arduino / ESP-IDF
  • WiFi 库 WiFi.h
  • MQTT 库 PubSubClient
  • JSON 处理 ArduinoJson
  • 存储 Preferences (NVS)

通信协议

  • WiFi 标准 802.11 b/g/n 2.4GHz
  • 物联网协议 MQTT 3.1.1
  • I2C 总线 400kHz (OLED)
  • 单总线 1-Wire (DS18B20)
  • ADC 采样 12-bit 0-3.3V

数据流程图

1

传感器数据采集

DS18B20 温度传感器通过 1-Wire 总线读取环境温度（GPIO 4）；光敏电阻通过 ADC 通道读取光照强度（GPIO 34），数据经过滤波处理后得到有效值。

2

本地显示与逻辑判断

采集的数据通过 I2C 总线发送到 OLED 显示屏实时显示（GPIO 21/22）；同时与预设阈值比较，判断是否触发报警或继电器动作。

3

WiFi 网络连接

ESP32 通过 WiFi 模块连接到无线路由器，建立 TCP/IP 网络连接。配置信息存储在 NVS 非易失性存储器中，支持断线自动重连。

4

MQTT 数据上报

将传感器数据封装成 JSON 格式，通过 MQTT 协议发布到云平台的指定主题（pct100/{device_id}/upload），供远程监控系统订阅。

5

远程控制响应

订阅云平台下发的控制主题（pct100/{device_id}/control），接收远程指令并执行相应操作（开关继电器、修改参数等）。

温度采集模块 (ds18b20.cpp)

负责 DS18B20 传感器驱动，支持单总线通信，温度数据读取和转换。

GPIO 4

光照采集模块 (adc.cpp)

负责 ADC 模数转换，读取光敏电阻模拟信号并转换为光照强度值（Lux）。

GPIO 34 (VP)

OLED 显示模块 (oled.cpp)

负责 OLED 屏幕驱动，I2C 通信，显示传感器数据、网络状态、系统信息等。

GPIO 21/22

按键处理模块 (key.cpp)

负责独立按键扫描，支持短按/长按识别，消抖处理，触发相应功能。

GPIO 15

LED 控制模块 (led.cpp)

负责状态指示灯控制，温度/光照超限报警提示。

GPIO 2

继电器模块 (relay.cpp)

负责继电器开关控制，可联动外部设备，支持自动/手动模式。

GPIO 5

WiFi 连接模块

负责 WiFi 网络连接，AP/STA 模式配置，断线重连，信号强度检测。

板载天线

MQTT 通信模块

负责 MQTT 协议实现，数据上报，指令订阅，心跳保持，QoS 管理。
